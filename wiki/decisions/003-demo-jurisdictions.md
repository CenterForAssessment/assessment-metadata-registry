# ADR-003: Demonstration Jurisdictions + Comparability Schema Extension

**Status:** Accepted
**Date:** 2026-07-02

## Context

The registry needed a second (and third) jurisdiction to exercise multi-jurisdiction
behavior and the historical features the design targets (scale transitions, gaps). SGPc's
`testSGPc()` already defines two demonstration jurisdictions and builds their metadata
*inline* (`sgpc_test_scenarios()` / `sgpc_test_metadata_records()`):

- **State D (`SD`, `sd-summative`)** — a vertical scale with a COVID gap (no 2020
  administration); MATHEMATICS + READING, grades 3-8; proficiency_boundary targets.
- **State C (`SC`, `sc-summative`)** — a mid-window scale transition; ELA + MATHEMATICS,
  grades 3-8; explicit scale_score ("ELP-like exit") targets.

Because these are demonstration data, we are free to invent values — an ideal way to
exercise schema surface that real states expose messily.

## Decision

**1. Author SD and SC into the registry** as real sidecars (assessment + accountability),
generated by `tools/seed_demo.py`, mirroring the testSGPc scenarios and applying ADR-002
(targets in accountability records). Using demonstration freedom, we represent the
structural features faithfully rather than flattening them as the inline builder does:

- **SD** omits a 2020 sidecar (the COVID gap is a first-class absence) and records
  `comparability.administered`/notes on the gap-adjacent year.
- **SC** authors a genuine scale break: a *legacy* scale in 2013-2014 and a *new*,
  non-comparable scale from 2015 (different cutscores and exit thresholds), so the
  changelog gets real cutscore-change and target-change events.

**2. Extend `amr.assessment_system.v1` with an optional `comparability` block**
(`comparable_to_prior_year`, `scale_transition`, `prior_scale_name`, `administered`,
`notes`). ADR-000 scoped comparability as in-scope; this is where it lands. It is
projected to `build/index.json` and a `comparability` SQLite table, and surfaced in R via
`amrr::amrr_comparability()`.

## Consequences

- Registry holds 3 jurisdictions (IN, SC, SD); 48 sidecars validate.
- The changelog is now exercised with real signal: SC's 2014->2015 scale break (cutscores
  + exit target) and the ILEARN COVID gap (its accountability target disappears in 2020,
  returns in 2021).
- A **changelog fix** landed: accountability target diffs were keyed incorrectly
  (`pm.get(sid)` instead of `pm.get((sid, content_area))`) and silently reported no
  changes; now fixed and covered by the demo data.
- `amrr` gains `amrr_comparability()`; a demo testthat suite covers a second jurisdiction,
  a scale_score exit target on a summative, and the comparability surface.

## Relationship to testSGPc (future wiring)

`testSGPc()` still builds its metadata inline. A later SGPc-side change can have it consume
these registry sidecars via `amrr::get_metadata()` (the [[sgpc-registry-consumption-contract]]).
The demo cutscores/targets here match the inline values for the post-transition period;
the invented legacy-scale years (SC 2013-2014) are additive demonstration history. Wiring
testSGPc will reconcile any remaining differences and update its baselines.
